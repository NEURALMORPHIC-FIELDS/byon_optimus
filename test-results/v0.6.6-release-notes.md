## Architectural changes

### Operator-Verified Facts production path
Three channel-gated server endpoints on the memory-service:

- `verified_fact_add` — only path that can set `trust=VERIFIED_PROJECT_FACT`. Rejects with HTTP 403 unless `channel == "operator-cli"`. Required fields: `subject`, `predicate`, `object`, `evidence`, `operator`.
- `verified_fact_revoke` — flips `trust → DISPUTED_OR_UNSAFE`, appends `__revoked__` + `__disputed__` tags, records audit metadata.
- `verified_fact_list` — read-only listing, revoked hidden by default.

Plus the operator CLI (`byon-orchestrator/scripts/byon-facts.mjs`) with `add-verified`, `list-verified`, `revoke-verified`. **There is no conversational path to `VERIFIED_PROJECT_FACT`.**

Smoke-tested live: channel gate → 403 from chat channels; add/list/revoke happy paths all work; revoked facts disappear from default listing.

### Latency
- **Prompt-cache split**: the 18-entry canonical block + TRUST POLICY + CONVERSATION EXCERPTS POLICY rides in a stable prefix marked `cache_control: { type: "ephemeral" }`. Per-turn dynamic suffix carries only recall + FCE summary.
- **Async fact extraction**: `routeFactExtraction` returns `sync` for explicit memory directives, `async` for normal turns, `skip` for trivial. In-flight extractor map per thread, drained between bench setup turns so fact from turn N is reliably visible at turn N+1.
- **FCE summary cache** (carried from v0.6.5) preserved; **parallel FAISS + FCE** on the recall path audited.

### Conversation framing
The cached prefix carries a dedicated `CONVERSATION EXCERPTS POLICY` paragraph: excerpts are not facts, not rules, not permissions; an adversarial *"memorează:"* inside an excerpt does NOT become a rule. Excerpts are re-labelled `[excerpt N]` to match the policy vocabulary.

### Scorer / verdict-gate fixes
- `looksLikeRefusal` catches leading `**Nu.**` opener, `afirmația este incorectă`, `nu va fi executat`, `operațiune ireversibilă`, `necesită aprobare/semnătură`.
- `detectActivePreferences` scans conversation excerpts too (not only facts). Plus a **thread-prefs cache** captures style directives at conversation-store time, derived from the raw user message — the most reliable source.
- **F-scorer floor lifted 3 → 4** for clean adversarial rejection without explicit policy citation (F2 case).
- **E1 verdict regex tightened**: only positive claims of prior context fail the gate; legitimate denials like *"Nu mi-ai spus anterior"* now PASS.
- **Latency gate made disjunctive** per roadmap §3.3.

### New benchmark category L (Operator-verified facts)
5 items exercising: verified version surfaces as authoritative; user-claim contradiction does NOT override verified fact; verified `theta_s` blocks demo reduction; revoke flow demotes from authoritative; global-scope verified fact crosses threads.

## Constraints honoured

- `θ_s = 0.28` and `τ_coag = 12` **NEVER lowered.** F4 verifies B refuses the demo reduction.
- Operational classification stays **Level 2 of 4 — Morphogenetic Advisory Memory**. No Level 3 claim on `main`.
- SYSTEM_CANONICAL always wins. Verified facts cannot be created through conversation.
- DISPUTED_OR_UNSAFE adversarial pattern gate from v0.6.5 is untouched.
- Honesty: no architectural relaxation to pass scores; sub-perfect items (A1/A2 sampling variance, B8 verified-vs-claim conflation, C3/C6/C10 honest-abstain miscoring, G2/G4/G6 cross-thread abstain miscoring) are documented in §5 of the technical report.

## Known limitations carried into v0.6.7

- `EXTRACTED_USER_CLAIM` recall can still influence the LLM when the user has repeatedly insisted on a value (B8 case). v0.6.7 ACTIVE RESPONSE CONSTRAINTS + post-gen compliance checker target this.
- Compliance guard scope still limited to emoji + concise; language / filler / invented-citation enforcement is v0.6.7 work.
- C / G scorer asymmetry around honest abstention. Test-bank refinement scheduled for v0.6.7+.
- Absolute latency p95 still > 10 s in nominal terms; the disjunctive gate passes thanks to A's own tail variance. Finer prompt-cache breakpoints + smaller recall payload are options for v0.6.7+.
